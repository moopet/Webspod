Console commands:

/beepplays the beep sound associated with the current theme.
/cloneopens a duplicate window to the same host, with all the same settings.
/commandslists all available console commands.
/connect <world>
/connect <hostname> <port>		connects to a talker. If there is an existing connection to a talker, it is closed first. 'world' may be any talker name or partial talker name in the world list. e.g. '/connect surfers' or '/connect surfers.org 4242'
/disconnect
/dc		disconnects from the remote talker.
/helpdisplays this page.
/listthemeslists all available themes.
/listworlds [-s|--short] [string]lists world definitions. Use of the '-s' parameter lists world names only. If 'string' is included, only worlds whose names include that string are listed.
/mark [text]draws a horizontal bar across the window as a place marker, with optional text notes.
/reconnectdisconnects and immediately reconnects to the current talker, keeping all your webspod settings intact.
/repeat [count]repeats the last command (not IRC or console command) count times.
/set <option> [=] <on|off|[value]>  toggle one of the session variables. Available variables:
colours, debugging, ircemu, localecho, smilies, sounds, theme, timestamps.
/spodlist <user>opens the user's details page on spodlist.org.
/statusdisplays system settings, including session variables.
/telnet <hostname> [port]connect to a line-based telnet host. The telnet login port is used if port is omitted. Note: this command may not be available depending on the way the program has been configured.
/versiondisplays version and build date.
/world <world>
/world <hostname> <port>		this command is simply an alias for /connect.

IRC emulated commands:

/ignore <user>toggles ignoring a specified user.
/join #<channelname>joins a channel, closing your connection to any previous channel. If the new channel name does not exist, it will be created.
/listcurrently not supported.
/meemotes an action.
/motddisplays the talker's message of the day.
/msg <user> <message>sends a private message to the specified user.
/nameslists all the users on the server.
/nickcurrently not supported.
/notice <user> <message>sends a notice to the specified user.
/partdisconnects you from the channel you were on.
/query <user> <message>sends a private message to the specified user. Currently just an alias for "/msg".
/quit <quit message>quits the session and logs you off, with a custom disconnect message where supported.
/say <message>sends a message to the current room.
/timetells you the time as set on the current server.
/timestamp [on|off]aliased to "/set timestamps on".
/topic [new topic]displays or sets the current topic.
/whoisbrings up information on another connected user.
/whowasbrings up information on another user who is no longer connected.

FAQ:

Why won't it do anything at all? Make sure JavaScript is enabled in your browser. It won't work without it yet.
If you're running ad blocking software like popupstopper, then you will have to uncheck the "new window" option on the connection page or turn off the other software.
The IRC commands don't do what I expect. All talkers don't use the same command set. This emulator turns IRC commands into their ewtoo/pg+ style equivalents.
The IRC commands claim they're not recognised. IRC emulation mode is probably turned off. Try "/set ircemu on".
Why aren't some lines being displayed? They are. They're just in a colour that's the same as the background. Most talkers will let you configure how they output colour, or turn it off completely for just this reason. Try the "setcolor" and "color" commands.
What happens if I enable IRC emulation, enter converse mode and try to use the editor? Bastard.
I found a bug; your code sucks! Have a medal. send me an email about it and I'll do my best to fix it.
If you want to find out more about talkers, try looking at ewtoo.org or spodlist.org.

Overview:

Webspod is an HTML interface for connecting to talkers. You run it on a webserver and then browse to it to spod. It is free software, released under the GNU General Public License, the details of which can be found here.
It was written from scratch by Ben Sinclair (moopet) and is copyright © 2003. It bears similarities to other web connection interfaces (balloon's HTMLCon which can be found at caves.uk.net and the Uberworld webclient from uberworld.org)
It can be started directly from your own webpage. Here is an example page.

Features:

  • Supports scriptable, stylesheet-based "themes" including sounds.
  • Optionally converts ASCII smilies to graphical icons.
  • Optionally timestamps every line received from the talker.
  • Optionally echoes user input to the main window.
  • Optionally converts IRC-commands to ew-too style commands on-the-fly.
  • Access to all options is available through the transparently embedded console.
  • Complete self-contained client, allows disconnection and reconnection to other hosts.
  • Auto-highlights any embedded URLs.
  • Multi-session safe (multiple tabs in mozilla work great too!)
  • Includes easy provision for hard-coding talker address into the cgi
  • Can extract current talker addresses and details from list.ewtoo.org
  • Can look up players' profiles on spodlist.org
  • Includes a comprehensive debugging mode.

Benefits:

  • Allows spods to connect to talkers from behind facist firewalls.
  • Allows partial connection to talkers via SSL if the webserver is running it.

Drawbacks:

  • It's still quite beta.
  • It uses a lot of non-standard HTML tricks to make things work.
  • Though it'll work if your firewall allows outbound web requests, it almost certainly won't work through web proxy servers.
  • Webspod is itself a proxy service. Every connection routed through webspod will appear to be coming from the same IP, that of the server. This could give the talker's admin a headache for sitebans, etc.
  • Each connection will launch a new task on the server Currently this eats up about 55k or memory per connection :(
  • ident will not be provided from the user's local machine, and if identd is running on the server hosting webspod, all talkers will get will be something like "www-data".
  • Players must trust the service hosting site not to harvest passwords as all text is processed by these CGIs.
  • Might make your web server's access logs a bit hefty.

Known bugs:

  • Mozilla doesn't always play sounds properly.
  • IE doesn't correctly render the javascript for opening a window with "/clone".
  • ident checking is currently disabled due to hanging on some people's connections.
  • Some stylesheets including bitmap backgrounds slow the interface down unacceptably.
  • In mozilla, especially, the service slows down after a couple of hours due to the streamed page getting stupidly large.
  • Ewe-editor mode is pretty shoddy.
  • Doesn't forward unknown console commands to the talker in IRC emulation mode.
  • Sometimes doesn't scroll the last line output.
  • Aqua theme is broken slightly due to different renderings in different browsers.
  • Sometimes it doesn't keep receiving data from the talker without hitting enter a lot???
  • Auto-focus script puts cursor at start of command.
  • Sometimes it disconnects from a talker immediately after opening the port.

Changelog:

"development" version 1.2 beta

  • Optional swear filter (for received text only, and without context).
  • New, completely awful 'kids' theme.
  • Completely removed the stealth command functions.
  • Added syslogging of errors and connect/disconnects.
  • Added read-only /wsctl command to console.
  • Fixed timestamper bug.
  • Added command aliases.
  • Took the session variables display from /status and put it in /set.
  • Moved most of the signal handlers into one general handler function.
  • Started trapping error signals. Eg, system now exits cleanly on a segfault.
  • Added auto-focus javascript for the command prompt (IE only, so far)
  • Fixed disabled command prompt theme glitch.
  • Added a sighup handler to cleanly quit the program with a "we're now rebooting" type message.
  • Added better disconnection code. Terminates connection straight away rather than just waiting for a ping/sigpipe.
  • Removed the (so far) pointless /splay mIRC-mimicked command.
  • Converted the bell() routine to a more generic play_sound() function. Added a bell() macro. Added code for sounds in Mozilla (dodgy)
  • Fixed input frame bug and enlarged input buffer to 256-characters.
  • Added other options (hinames, debugging, colours) to index pages.
  • Fixed buffer wrap bug which broke colour codes.
  • Completely changed the command matching code to a more modular format.
  • Added screenshots to the website :)
  • Added support for javascript in themes.
  • Fixed local echo bug.
  • Vastly improved debugger.
  • Added individual bell wavs to each theme.
  • Added /repeat command.
  • Fixed the login/password blanking feature.

"development" version 1.10 beta

  • Added the hinames variable to allow on-the-fly highlighting of your username.
  • Added the mastodon theme, and improved the aqua theme.
  • Added /commands and /listworlds commands.
  • Changed some commands to give a feel closer to TF.
  • Added feature to let the /connect command look up talker names.
  • Improved the login/password blanking feature.
  • Moved the pinger code from input.cgi to a new file, pinger.cgi, to make the code a bit more efficient.
  • Added an error page for if you click connect and don't have a browser that does JavaScript.
  • Added a SIGPIPE handler to disconnect properly.
  • Removed "webshell" mode. That may well become a seperate project, a bit like port42.
  • Made the static HTML pages and stylesheets W3C compliant. This is currently impossible with the dynamic content.
  • Tidied code, changed stupid naming conventions to they all follow a Plan. More documentation.
  • Added browser identification to limit options like sounds, etc.
  • Changed command parsing code. /set shortcuts no longer available.
  • Added experimental ewe-editor mode support to IRC emulation routines.
  • Added script to the main page to automatically, politely, download a new talker list file if one doesn't exist - for example, if the system has been restarted and /tmp has been wiped.
  • Added connection time indicator to wsctl.
  • Added talker name lookups for window titling.
  • Added onBeforeUnload() confirmation (IE 4.0+ only) to stop IE windows accidentally being replaced by other applications.
  • Fixed hard-coded talker form bug where it simply refused to open a connection window.

"stable" version 1.0d (and previous)

  • No changes recorded.
